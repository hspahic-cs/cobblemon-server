# server-end-spawn-nerfs

Datapack override that makes the **Beldum line rare in the End dimension**.

## Why

Beldum / Metang / Metagross spawn in the End (`#cobblemon:is_end`) at the **rare**
bucket — `0.1` bucket weight, i.e. ~100× more common than the legendaries and ultra
beasts, which the ATM spawn-nerf pipeline (`ops/gen_spawn_nerfs.py`) already pins to the
**ultra-rare** bucket (`0.001`). End exploration was turning up Metagross/Metang far too
often relative to the actual End end-game targets.

## What it does

Overrides Cobblemon's base spawn files:

- `data/cobblemon/spawn_pool_world/0374_beldum.json`
- `data/cobblemon/spawn_pool_world/0375_metang.json`
- `data/cobblemon/spawn_pool_world/0376_metagross.json`

For each, **only the `#cobblemon:is_end` spawn entry** has its `bucket` changed from
`rare` → `ultra-rare`. Weights and every other biome entry (Aether, dripstone, peak,
Bumblezone howling constructs) are left exactly as Cobblemon ships them, so the line is
still obtainable at normal rates everywhere except the End.

These are full-file overrides (datapacks can't patch a single spawn entry), regenerated by
copying the base file from the Cobblemon jar and flipping the one bucket. If Cobblemon
changes these base spawns upstream, re-copy and re-apply the single edit.
