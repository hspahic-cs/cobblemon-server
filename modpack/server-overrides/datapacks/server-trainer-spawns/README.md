# server-trainer-spawns

Fixes the RCTmod **wild trainer pool at high player levels**. Above party level ~60 the pool used to
collapse to jar-default *boss* trainers (gym leaders, Elite Four, champions, rivals, battleground,
team admins) — competitive teams, often over-leveled (rivals/champs/E4 are "key trainers" that bypass
the level-diff scaling, so they spawned 20+ levels above the player).

Three parts (all generated by `ops/gen_highlevel_trainers.py`):

- **A — remove gyms + Elite Four from wild:** every jar mob with `type` ∈ {leader, e4} → `spawnWeightFactor: 0`.
- **B — level-gate + thin the bosses:** champ/rival/battleground/ligh_of_ruin (and high-level team
  admins) → retyped to `normal` (respect scaled `maxLevelDiff`, no more over-leveled spawns) and
  `spawnWeightFactor` cut to `0.08` so they're drowned out by the new normal trainers.
- **C — add ~80 themed mid-tier trainers (L55–100):** 8 archetypes (Bird Keeper, Bug Catcher, Hiker,
  Fisherman, Black Belt, Psychic, Hex Maniac, Cooltrainer), 2–4 fully-evolved Pokémon, valid abilities
  from Cobblemon species data, **no movesets** (Cobblemon auto-assigns natural level-up moves — not
  competitive), empty EVs, moderate IVs, `type: normal`, `spawnWeightFactor: 0.35`, series matched to
  the existing wild pool so they actually spawn.

Regenerate after an RCTmod version bump: re-run the script (it re-reads the jar mob defs).
